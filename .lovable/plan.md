## เป้าหมาย

ปรับหน้า `/dashboard` 2 เรื่อง:

1. **เลิกใช้ Tabs** — แสดงทุกหมวดของรายงานเรียงต่อกันลงมาเลย ไม่ต้องกดสลับ
2. **เพิ่มรายงานรายพนักงาน** — แต่ละคนทำขั้นตอนอะไรไปบ้าง กี่งาน พร้อมฟิลเตอร์ครบ

## โครงหน้าใหม่ (เรียงจากบนลงล่าง)

```text
[Header + ฟิลเตอร์ (หมวดหมู่ / พนักงาน / ขั้นตอน / ช่วงเวลา) + ปุ่ม Export]
[การ์ดสรุป 3 ใบ: เสร็จวันนี้ / เสร็จเดือนนี้ / บันทึกทั้งหมด]

1. ภาพรวม 14 วัน — เริ่มงาน vs เสร็จงาน (BarChart)
2. กิจกรรมรายขั้นตอน 30 วัน (PieChart)
3. อันดับพนักงาน — งานที่ทำเสร็จ (BarChart แนวนอน)
4. ★ รายงานรายพนักงาน × ขั้นตอน (ใหม่)
5. เปรียบเทียบเดือนนี้ vs เดือนก่อน (MoM table)
6. งานที่เกินเวลามาตรฐาน
7. กิจกรรมล่าสุด (30 รายการ)
```

ฟิลเตอร์ด้านบน (หมวดหมู่ / พนักงาน / ขั้นตอน / รายวัน-รายเดือน + วันที่) มีผลกับทุกเซกชันที่เกี่ยวข้องเหมือนเดิม

## เซกชันใหม่: รายงานรายพนักงาน × ขั้นตอน

ใช้ข้อมูลจาก `filtered` (โดน scope วัน/เดือน + ฟิลเตอร์หมวด/พนักงาน/ขั้นตอน) แล้วจัดกลุ่มเป็น matrix:

- แกนแถว = พนักงาน
- แกนคอลัมน์ = ขั้นตอน (เฉพาะที่มีงานในช่วงเวลาที่เลือก)
- ค่าในเซลล์ = จำนวนงาน (นับ unique `job_id` ที่ `action = "finish"` ของพนักงานคนนั้นในขั้นตอนนั้น)
- คอลัมน์สุดท้าย = "รวม" (ผลรวมทุกขั้นตอนของแถว)
- แถวสุดท้าย = "รวม" ต่อขั้นตอน

ใต้ matrix จะมีกล่องขยายรายคน (Accordion / รายการการ์ด) แสดง:
- ชื่อพนักงาน + จำนวนงานรวม
- breakdown รายขั้นตอน: `ขั้นตอน A — 5 งาน · เวลาเฉลี่ย 12.4 น.`
- เวลาเฉลี่ยต่อขั้นตอนคำนวณจาก `sessions` (start→finish pairs) ที่อยู่ใน scope

จะ render เฉพาะพนักงาน/ขั้นตอนที่มีข้อมูล เพื่อให้ตารางไม่เตียน

- ฟิลเตอร์บาร์ด้านบน (เก็บไว้เหมือนเดิม)
